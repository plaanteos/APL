# APL — Dental Practice Management Platform

Full-stack web application for commercial and administrative management of dental practices. Built to centralize order tracking, client management, financial balances, and direct communication channels in a single admin interface.

**Live:** [administracionapl.netlify.app](https://administracionapl.netlify.app)

---

## Tech Stack

| Layer | Technology |
|---|---|
| Frontend | React + Vite + TypeScript |
| Backend | Node.js + Express + TypeScript |
| ORM | Prisma |
| Database | PostgreSQL (external — Neon / Supabase) |
| Auth | JWT (auto-generated secret) |
| Email | SMTP (Gmail) / Resend (recommended for production) |
| WhatsApp | Multi-session architecture via WhatsApp Web |
| Deploy — Frontend | Netlify |
| Deploy — Backend | Render |

## Features

- **Dashboard** — real-time metrics: total orders, revenue, active clients, pending orders
- **Order management** — create, track status, and access individual order balances
- **Client management** — full CRUD, balance history, direct contact via WhatsApp or email
- **Balance module** — payment tracking, delivery confirmation, Excel export
- **WhatsApp integration** — multi-session architecture; each admin connects their own number via QR
- **Email integration** — SMTP / Resend for transactional messaging
- **Authentication** — JWT-based login with password management
- **Audit layer** — normalized DB schema (3NF) with triggers and audit tables

## Local Setup

### Prerequisites

- Node.js LTS
- PostgreSQL instance (local or external: Neon, Supabase, etc.)

### Backend

```bash
cd backend
npm install
cp .env.example .env
# Edit .env — set DATABASE_URL, JWT_SECRET, SMTP credentials
npm run db:generate
npm run db:migrate
npm run dev
```

### Frontend

```bash
cd figma
npm install
cp .env.example .env
# Edit .env — set VITE_API_URL
npm run dev
```

### Clean workspace (remove generated artifacts)

```bash
# Remove dependencies and builds
rm -rf **/node_modules **/dist **/build backend/logs
# Note: never commit .env files — use .env.example as template
```

## Deploy

### Backend — Render

The `render.yaml` config handles the backend deploy automatically:

```yaml
buildCommand: npm ci --include=dev && npm run build && npx prisma generate
startCommand: npm start
healthCheckPath: /health
```

> ⚠️ Do **not** run `prisma migrate` during the build step — if the DB is suspended it will cause a `P1001` error. Run migrations manually from your local machine with a valid `DATABASE_URL`.

Required environment variables (set in Render Dashboard):

| Variable | Description |
|---|---|
| `DATABASE_URL` | PostgreSQL connection string |
| `JWT_SECRET` | Auto-generated by Render |
| `FRONTEND_URL` | `https://administracionapl.netlify.app` |
| `SMTP_HOST` | `smtp.gmail.com` |
| `SMTP_PORT` | `587` |
| `SMTP_USER` | Sender email address |
| `SMTP_PASS` | App password (set as secret) |
| `RESEND_API_KEY` | *(optional)* Resend API key for production |

### Frontend — Netlify

Configured via `netlify.toml`. Set `VITE_API_URL` pointing to your Render backend URL in Netlify's environment variables.

## WhatsApp Architecture

Multi-session design: each admin connects their own WhatsApp number via QR scan. Sessions are stored per `user_id` and isolated at the database level.

```
Admin1 → Backend → WhatsApp Session (Admin1) → Client
Admin2 → Backend → WhatsApp Session (Admin2) → Client
```

See [`whatsapp_multiuser_architecture.md`](./whatsapp_multiuser_architecture.md) for full design documentation.
